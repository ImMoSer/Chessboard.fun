# Ретроанализ партии (Retro Analysis) — Отчет о разработке

## Обзор задачи
Реализована функция «Ретроанализа» для шахматного приложения, позволяющая пользователю получить детальный отчет о качестве его игры после завершения партии (в частности, в режиме `/opening-sparring`).

## Что было сделано

### 1. Архитектура и сервисы
- **RetroAnalysisService.ts**: Создан изолированный сервис для управления движком Stockfish.
    - **Алгоритм "Backward Analysis"**: Анализ идет от конца партии к началу. Это позволяет максимально эффективно использовать хеш-таблицу (Transposition Table), так как оценки финальных позиций помогают быстрее оценивать предшествующие.
    - **Изоляция**: Используется отдельный воркер Stockfish, чтобы не мешать основному анализу в реальном времени.
    - **Конвертация**: Реализована автоматическая конвертация UCI-ходов движка в SAN-нотацию для удобства пользователя.

### 2. Хранение данных (State Management)
- **retroAnalysis.store.ts**: Pinia-стор для управления состоянием анализа.
    - Хранит настройки (Threads, Hash, Depth, Movetime).
    - Отслеживает прогресс (0-100%).
    - Хранит итоговый объект `report`.

### 3. Пользовательский интерфейс (UI)
- **RetroAnalysisModal.vue**:
    - Модальное окно с настройками мощности (Threads — 50% от доступных по умолчанию, Hash, лимиты по времени и глубине).
    - Визуализация прогресса (круговой индикатор).
    - Итоговая карточка с точностью (Accuracy %), средней потерей сантипашек (ACPL) и счетчиком ошибок.
- **Интеграция в OpeningExamView**: Добавлена кнопка вызова анализа, которая появляется только после завершения или прерывания игры.

### 4. Визуализация и логика "Для игрока"
- **Фокус на игроке**: Система автоматически определяет цвет игрока (`playerColor`).
    - Статистика (точность и кол-во ошибок) считается **только для ходов игрока**.
    - Маркеры в PGN (??, ?, ?!) отображаются **только на ходах игрока**.
- **AnalysisPanel.vue**: 
    - В список PGN добавлены маркеры классификации ходов.
    - Добавлена карточка «Совет», которая появляется при клике на ошибочный ход. Она показывает, какой ход был лучшим по мнению Stockfish и как изменилась оценка.

### 5. Технические детали
- **Движок**: Stockfish NNUE (WASM).
- **Типизация**: Проект полностью очищен от ошибок типов (`pnpm type-check` проходит без ошибок).
- **PgnService**: Расширен методом `getMainlineNodes()` для получения всех позиций основной линии.

## Текущий статус
Функционал полностью готов к использованию и протестирован. Анализ запускается, успешно проходит в обратном порядке и выдает корректный отчет.
